### Authorship Policy & Guidelines

This policy ensures transparent credit, accountability, and equity in the project:

#### Core Authors
- **Zakir Bayramov** and **Gustav Olaf Yunus Laitinen-Lundstrom** are designated as core authors
- Both must remain listed in:
  - `AUTHORS.md` (this file)
  - `CITATION.cff` (machine-readable metadata)
  - `pyproject.toml` (Python package metadata)
  - GitHub repository settings
- Changes to authorship require explicit pull request with rationale

#### Significant Contributors (Potential Co-Authors)
Contributors with any of the following may be considered for authorship:
- **10+ commits** with meaningful contributions
- **Major feature implementation** (>500 lines, primary responsibility)
- **Research contribution** (algorithm design, theoretical work)
- **Publication co-authorship** (joint research papers)

#### New Authorship Request Process

If you believe you qualify for authorship, follow this process:

1. **Open Discussion** — Open a GitHub Discussion (not Issue) with:
   - Summary of contributions
   - List of commits/features
   - Rationale for authorship request

2. **Core Team Review** — Both Zakir and Gustav review and discuss
   - Assessment of contribution scope
   - Impact on project direction
   - Fit with project goals

3. **Consensus Decision** — Both core authors must agree
   - Decision documented in Discussion
   - If approved: new author added to metadata files
   - If declined: feedback provided for future consideration

4. **Recognition** — Approved authors listed in:
   - AUTHORS.md (with profile)
   - CITATION.cff
   - Release notes for version where authorship added
   - GitHub repository contributors

#### Authorship vs. Acknowledgment

**Authors** have contributed substantially to:
- Algorithm design, implementation, and validation
- Architectural decisions and code organization
- Research direction and methodology
- Multi-year commitment to the project

**Contributors** (acknowledged but not co-authors) have:
- Fixed bugs or improved documentation
- Contributed features or smaller components
- Provided feedback or testing
- Made time-limited contributions

We recognize both equally in GitHub Contributors and release notes.

### Contributing to the Project

We welcome contributions! To contribute:

1. **Check Contributing Guide** — See [CONTRIBUTING.md](/CONTRIBUTING.md)
   - Development environment setup
   - Code style standards
   - Testing requirements
   - Pull request process

2. **Fork Repository** — Create your working copy
   ```bash
   git clone https://github.com/YOUR_USERNAME/stochastic-transit-router.git
   cd stochastic-transit-router
   ```

3. **Create Feature Branch** — Use descriptive branch names
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/issue-123-short-description
   ```

4. **Make Changes** — Implement your feature/fix
   - Add tests for new functionality
   - Update documentation as needed
   - Follow code style guidelines
   - Run `make lint`, `make type-check`, `make test`

5. **Submit Pull Request** — Against `main` branch
   - Clear description of changes
   - Reference related issues (e.g., "Closes #123")
   - Ensure CI/CD checks pass
   - Request review from maintainers

6. **Respond to Feedback** — Collaborate with reviewers
   - Address code review comments
   - Update PR as needed
   - Respond within 5 business days

7. **Merge & Celebrate** — PR merged by maintainer
   - Changes included in next release
   - You're listed as contributor
   - Thank you for contribution!

### Types of Contributions We Welcome

- **Code Improvements:** Bug fixes, performance optimization, feature implementation
- **Documentation:** Clarifications, examples, translations, corrections
- **Testing:** Bug reports with reproduction steps, test cases, edge cases
- **Review:** Code review feedback, quality suggestions
- **Design:** UX improvements, API design suggestions, architecture feedback
- **Research:** Algorithm improvements, theoretical contributions
- **Infrastructure:** CI/CD improvements, deployment optimizations, tooling
